# UK Renewable Energy Forecasting
> Forecasting solar and wind energy output for London using SARIMA and Prophet — built as part of a Data Science internship application to Shell.

---

## Overview

This project builds and compares two time-series forecasting models — **SARIMA** and **Facebook Prophet** — to predict daily solar and wind capacity factors for London, UK.

A key insight explored throughout is the **complementarity** of solar and wind: solar peaks in summer while wind peaks in winter, meaning a combined portfolio produces more stable energy output year-round. This is directly relevant to Shell's renewable energy strategy.

The project ends with an **interactive Streamlit dashboard** allowing users to explore forecasts, seasonality patterns, and model performance side by side.

## Methodology

### Data Sources
| Source | Data | Coverage |
|---|---|---|
| [Renewables Ninja](https://www.renewables.ninja/) | Hourly solar & wind capacity factors | 2019–2023 |
| [Open-Meteo](https://open-meteo.com/) | Temperature, cloud cover, wind speed, radiation | 2019–2023 |

### Models

**SARIMA** `(2,0,2)(1,1,1,7)`
- Classical statistical model capturing short-term autocorrelation and weekly seasonality
- Parameters guided by ACF/PACF analysis in Phase 2
- Fitted using `statsmodels`

**Prophet**
- Decomposition-based model developed by Meta's data science team
- Handles yearly and weekly seasonality automatically
- Better suited to long-term trend and seasonal pattern capture
- Multiplicative seasonality mode used for solar; additive for wind

### Evaluation
Models evaluated on the held-out **2023 test year** using:
- **RMSE** (Root Mean Squared Error) — penalises large errors more heavily
- **MAE** (Mean Absolute Error) — average magnitude of error

---

## Key Findings

- **Solar and wind are inversely seasonal** — solar peaks June–August, wind peaks November–February. A combined portfolio reduces output variance by ~30% compared to either source alone.
- **Shortwave radiation** is the strongest predictor of solar output (correlation > 0.85)
- **Wind speed** is the strongest predictor of wind output, but wind is inherently harder to forecast due to higher short-term variability
- **Prophet outperformed SARIMA** on yearly seasonal patterns; SARIMA was more precise on short-term weekly cycles

## How to Run

### 1. Clone the repo
```bash
git clone https://github.com/YOUR_USERNAME/renewable-energy-forecasting.git
cd renewable-energy-forecasting
```

### 2. Install dependencies
```bash
pip install -r requirements.txt
```

### 3. Get a free Renewables Ninja API token
Sign up at [renewables.ninja](https://www.renewables.ninja/) → Profile → API access  
Paste your token into cell 3 of `phase1_data_collection.ipynb`

### 4. Run the notebooks in order
```
phase1_data_collection.ipynb  →  generates data/
phase2_eda.ipynb               →  generates EDA plots
phase3_modelling.ipynb         →  generates forecasts + metrics
```

### 5. Launch the dashboard
```bash
streamlit run dashboard.py
# or if streamlit is not on PATH:
python -m streamlit run dashboard.py
```
